# Foundation Header

`foundation-header` is a micro front-end and a semi-batteries-included component.
It consists of a navigation bar and a flyout menu, with routing, account-logout capabilities, and the ability to change the current language.

![Header with the standard genesis logo](/img/001_develop/03_client-capabilities/004_header/foundation-header-standard.png)

## Examples

### Adding a header component to the template

Here is an example of the navigation bar with three navigation items, and all three control buttons shown.

<Tabs defaultValue="genesis" values={[{ label: 'Genesis', value: 'genesis', }, { label: 'React', value: 'react', }, { label: 'Angular', value: 'angular', }]}>

<TabItem value="genesis">

Declaration:
```html
  <foundation-header
    show-luminance-toggle-button
    show-misc-toggle-button
    show-language-selector
    :routeNavItems=${x => x.navItems}
  ></foundation-header>
```

Usage:
```typescript
@customElement({
  name: 'header-example',
  template: html`
    <foundation-header

        name: "table",
        variant: "solid",

}
```

### Configuring a form using the UI schema

You can configure the appearance of the form using the UI schema (see below).

Here is an example UI schema for configuring a form:

<Tabs defaultValue="genesis" values={[{ label: 'Genesis', value: 'genesis', }, { label: 'React', value: 'react', }, { label: 'Angular', value: 'angular', }]}>

<TabItem value="genesis">

Declaration:
```html
<foundation-header
  design-system-prefix="rapid"
  resourceName="EVENT_INSERT_USER"
  :uischema=${(x) => x.uiSchema}
></foundation-header>
```

Usage:
```typescript
const uiSchemaExample = {
```

### Configuring a form using a JSON schema (client-side)

Instead of providing resourceName, you can hard-code the JSON schema on the client.

:::info
Use this when you want to avoid fetching metadata from the server, but be aware that it could get out of sync if metadata changes on the server.
:::

Here is an example of a JSON schema:

<Tabs defaultValue="genesis" values={[{ label: 'Genesis', value: 'genesis', }, { label: 'React', value: 'react', }, { label: 'Angular', value: 'angular', }]}>

<TabItem value="genesis">

Declaration:
```html
<foundation-header
  design-system-prefix="rapid"
  resourceName="EVENT_INSERT_USER"
  :uischema=${(x) => x.uiSchema}
  :jsonSchema=${(x) => x.jsonSchema}
></foundation-header>
```

### Pre-filling a form with data (optional)

The data attribute allows you to pre-fill the form with information. A common use case for this is when editing an existing record.

<Tabs defaultValue="genesis" values={[{ label: 'Genesis', value: 'genesis', }, { label: 'React', value: 'react', }, { label: 'Angular', value: 'angular', }]}>

<TabItem value="genesis">

Declaration:
```html
<foundation-header
  design-system-prefix="rapid"
  resourceName="EVENT_INSERT_USER"
  :data=${(x) => x.data}
></foundation-header>
```

Usage:
```typescript
const existingData = {
  USER_NAME: "Genesis",
  EMAIL: "example@genesis.global",
  PASSWORD: "secret_password",
  PASSWORD_CONFIRMATION: "secret_password",
};

@customElement({
  name: 'form-example',
  template: html`
    <foundation-header
      design-system-prefix="rapid"
      resourceName="EVENT_INSERT_USER"
      :data=${(x) => x.data}
    ></foundation-header>
  `,
})
export class HeaderExample extends GenesisElement {
  data = existingData;
}
```
