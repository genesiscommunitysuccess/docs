# Genesis foundation utils

The `@genesislcap/foundation-utils` package provides a collection of utility services and helpers designed to simplify common development tasks within Genesis applications. It aims to enhance productivity by offering reusable solutions for patterns and problems encountered across different projects.

:::info
For more detailed information on API and configurations, refer to the [API documentation](docs/api/index.md).
:::

## Features

The library encompasses a broad range of features, including:

- **Data** - Manages in-memory databases and data storage solutions.
- **Decorators** - Provides decorators for enhancing class functionalities, such as rendering on change.
- **Design System** - Holds the design system configurations and utilities.
- **Directives** - Offers Angular-like directives for DOM manipulations, including conditional rendering and synchronization functionalities.
- **ENV** - Contains utilities for environment variable management and detection of development environments.
- **Error** - Deals with error mapping and handling strategies.
- **Formatters** - Includes functions for formatting dates and numbers according to locale settings.
- **Logger** - A simple logging utility for debugging and information tracking.
- **Mappers** - Focuses on data transformation, especially for DTOs (Data Transfer Objects).
- **Mixins** - Provides mixins for common functionalities like lifecycle hooks and state management.
- **Observer** - Implements the Observer pattern for reactive programming.
- **Resource** - Defines types and interfaces for resource management.
- **Serializers** - Contains serializers, particularly for JSON data handling.
- **State** - Manages reactive state and provides utilities for state observation.
- **Styles** - Includes utilities for styling, dealing with colors, typography, and DOM styles.
- **UUID** - Utilities for generating and managing UUIDs.
- **Window** - Provides abstractions and utilities for window object manipulations.

## Installation

To include `@genesislcap/foundation-utils` in your project, add it as a dependency in your `package.json` file and follow your project's routine for dependency installation.

```json
{
  "dependencies": {
    "@genesislcap/foundation-utils": "latest"
  }
}
```
