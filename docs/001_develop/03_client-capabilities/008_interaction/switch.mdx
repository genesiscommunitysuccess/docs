# Switch

A form-connected implementation of a [switch](https://w3c.github.io/aria/#switch) component.

Use cases:
* Toggling visibility of a UI section
* Disabling & enabling features

```typescript
import { Component, CUSTOM_ELEMENTS_SCHEMA } from '@angular/core';
import { FormsModule } from '@angular/forms';
@Component({
  selector: 'my-root',
  template: `
    <rapid-switch
      name="isSelected"
      [checked]="isSelected"
      (change)="onSwitchChange($event)"
    ></rapid-switch>
  `,
  styleUrls: ['./my.component.css'],
  standalone: true,
  schemas: [CUSTOM_ELEMENTS_SCHEMA],
  imports: [FormsModule],
})
export class AppComponent {
  isSelected: boolean = false;
  onSwitchChange(event: Event) {
    const target = event.target as HTMLInputElement;
    this.isSelected = target.checked;
    console.log(this.isSelected);
  }
}
```

## API

Property and attribute binding examples for Genesis Component syntax.

| Attribute/Property | Type | Description |
|---|---|---|
|checked|`boolean`|The boolean value of whether the switch is checked. |
|readonly|`boolean`|Configures the component so the user cannot alter the state via the web browser.|
|disabled|`boolean`|Same functionality as `readonly`, and additionally adds visual opacity to the component.|
|form|`string`|Associates this component to a form. Form id needs to be passed. If no Id informed, then it will be associated with the ancestor form.|

### Examples (closing tag omitted):

### Slots

| Slot | Use |
|---|---|
| switch | The control element. |

### Parts

| Part | Use |
|---|---|
| label | The label text.|
| checked-indicator | The checked indicator.|
| checked-message | The label when the switch is selected.|
| switch | The input control.|
| unchecked-message | The label when the switch is not selected.|

### Fired Events

| Part | Use |
|---|---|
| click | Emits a click event when the checked state changes.<br/>Access the value on the event via `.target.checked`.|

### Listened Events

This component doesn't listen to any events.
