## Initialising and adding to a new application

The best way to add the Reconciliation component to a new application is to use GenX - see [GenX CLI](/docs/develop/development-environment/genx/) for more details.

For example, the following command initialises a new application called `myapp` and adds the Reconciliation component to it.

<Tabs
    defaultValue="windows"
    values={[
        { label: 'Windows', value: 'windows', },
        { label: 'Mac and Linux', value: 'mac-and-linux', }
    ]
    }>
    <TabItem value="windows">
        ```shell title="Powershell"
        npx -y @genesislcap/genx@latest init myapp -x --no-shell; npx -y @genesislcap/genx@latest add myapp -x -s genesislcap/pbc-reconciliation-seed
        ```
    </TabItem>

    <TabItem value= 'mac-and-linux'>
        ```shell title="Bash"
        npx -y @genesislcap/genx@latest init myapp -x --no-shell && npx -y @genesislcap/genx@latest add myapp -x -s genesislcap/pbc-reconciliation-seed
        ```
    </TabItem>
</Tabs>

## Adding to an existing application

To add the Reconciliation component to an existing application, run this command, which uses genx to install the components (change _app-folder-name_ to the name of your app folder):

<Tabs
    defaultValue="windows"
    values={[
        { label: 'Windows', value: 'windows', },
        { label: 'Mac and Linux', value: 'mac-and-linux', }
    ]
    }>
    <TabItem value="windows">
        ```shell title="Powershell"
        npx -y @genesislcap/genx@latest add app-folder-name -x -s genesislcap/pbc-reconciliation-seed
        ```
    </TabItem>

    <TabItem value= 'mac-and-linux'>
        ```shell title="Bash"
        npx -y @genesislcap/genx@latest add app-folder-name -x -s genesislcap/pbc-reconciliation-seed
        ```
    </TabItem>
</Tabs>
