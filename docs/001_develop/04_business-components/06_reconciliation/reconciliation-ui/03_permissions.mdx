Your users need to have to correct permissions to use different features of the Reconciliation component.

Right codes control access to these features. Here are the codes available:

<details>
    <summary>Right codes</summary>
    | Right code | Description |
    | --- | --- |
    | ReconciliationConfigView | Ability to view any reconciliation configuration resource. |
    | ReconciliationConfigInsert | Ability to insert reconciliation configurations |
    | ReconciliationConfigUpdate | Ability to update reconciliation configurations |
    | ReconciliationConfigDelete | Ability to delete reconciliation configurations |
    | ReconciliationDictionaryView | Ability to view any reconciliation dictionary resource |
    | ReconciliationDictionaryInsert | Ability to insert reconciliation dictionaries |
    | ReconciliationDictionaryUpdate | Ability to update reconciliation dictionaries |
    | ReconciliationDictionaryDelete | Ability to delete reconciliation dictionaries |
    | ReconciliationResultView | Ability to view any reconciliation result resource |
    | ReconciliationRun | Ability to run a reconciliation |
    | ReconciliationCancel | Ability to cancel a reconciliation |
</details>

<details>
    <summary>Default profile set-up</summary>
    | Profile name | Linked codes |
    | --- | --- |
    | RECONCILIATION_USER | All Right Codes |
</details>

For full details of how to set up permissions, see our page on [authorisation](/server/access-control/authorisation/#users-profiles-and-right-codes).
