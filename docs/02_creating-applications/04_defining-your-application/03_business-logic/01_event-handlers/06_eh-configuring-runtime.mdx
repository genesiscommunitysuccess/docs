There are two important files in your application that contain configuration information: 
-  _application-name_**-processes.xml**
- _application-name_**-service-definitions.xml



### Configuring in processes.xml

Here is an example of a request reply configuration in an application's **processes.xml** file:

```xml
  <process name="POSITION_EVENT_HANDLER">
    <groupId>POSITION</groupId>
    <start>true</start>
    <options>-Xmx256m -DRedirectStreamsToLog=true -DXSD_VALIDATE=false</options>
    <module>genesis-pal-eventhandler</module>
    <package>global.genesis.eventhandler.pal</package>
    <script>position-eventhandler.kts</script>
    <description>Handles events</description>
    <classpath>position-messages*,position-eventhandler*</classpath>
    <language>pal</language>
  </process>
```

For more information on the tags that can be set within the configuration for your application, follow this [link](/creating-applications/configure-runtime/processes-xml).

### Configuring in service-definitions.xml

Here is an example of the event handler service configuration:

```xml
  <service host="localhost" name="POSITION_EVENT_HANDLER" port="11002"/>
```

For more information on the attributes that can be set in the service definition configuration, follow this [link](/creating-applications/configure-runtime/service-definitions).
